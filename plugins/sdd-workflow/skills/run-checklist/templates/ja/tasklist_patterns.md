# Run Checklist用TaskListパターン

## 初期タスク作成

検証開始時に、検証するカテゴリごとにタスクを作成：

```
TaskCreate:
  subject: "テストレビューの検証 (CHK-5xx)"
  description: "テストレビューのチェックリスト項目に対してテスト実行とカバレッジ計測を行う"
  activeForm: "テスト実行中"
```

## タスクカテゴリ

| カテゴリ            | Subject テンプレート                    | activeForm                    |
|:-------------------|:----------------------------------------|:------------------------------|
| 要件 (1xx)          | 要件レビューの検証 (CHK-1xx)             | 要件をチェック中               |
| 仕様 (2xx)          | 仕様レビューの検証 (CHK-2xx)             | 仕様を検証中                   |
| 設計 (3xx)          | 設計レビューの検証 (CHK-3xx)             | アーキテクチャを分析中         |
| 実装 (4xx)          | 実装レビューの検証 (CHK-4xx)             | リンター実行中                 |
| テスト (5xx)        | テストレビューの検証 (CHK-5xx)           | テスト実行中                   |
| ドキュメント (6xx)  | ドキュメントレビューの検証 (CHK-6xx)     | ドキュメントをチェック中       |
| セキュリティ (7xx)  | セキュリティレビューの検証 (CHK-7xx)     | セキュリティスキャン実行中     |
| パフォーマンス (8xx)| パフォーマンスレビューの検証 (CHK-8xx)   | ベンチマーク実行中             |
| デプロイ (9xx)      | デプロイレビューの検証 (CHK-9xx)         | 設定を検証中                   |

## ステータス更新

### カテゴリ開始時

```
TaskUpdate:
  taskId: "{task-id}"
  status: "in_progress"
```

### カテゴリ完了時

```
TaskUpdate:
  taskId: "{task-id}"
  status: "completed"
```

## 実行フロー例

1. 全カテゴリのタスクを作成（pending）
2. 各カテゴリに対して：
   - タスクを in_progress に設定
   - 検証コマンドを実行
   - 結果を記録
   - タスクを completed に設定
3. 最終レポートタスクを作成
4. 検証レポートを生成
5. レポートタスクを完了に設定
