# フェーズ別実行ルール

## Phase 1: セットアップ

```
1. ディレクトリ構造を作成
2. 型定義ファイルを作成
3. 基本インターフェースを定義
4. テスト環境をセットアップ
```

**完了基準**: すべての基盤ファイルが作成され、テストが実行可能な状態

## Phase 2: テスト

```
1. tasks.md のテストタスクを確認
2. 各機能の失敗するテストを作成
3. テストを実行しRed（失敗）を確認
4. tasks.md の [x] をマーク
```

**完了基準**: すべてのテストケースが作成され、期待通りに失敗している状態

## Phase 3: コア実装

```
1. Phase 2 で作成したテストに対応する実装
2. テストをGreen（成功）にする実装
3. 最小限の実装（過剰実装を避ける）
4. tasks.md の [x] をマーク
```

**完了基準**: すべてのコアテストが成功

## Phase 4: 統合

```
1. モジュール間の統合テストを先に作成
2. 統合テストを実行しRedを確認
3. 統合部分を実装してGreenにする
4. tasks.md の [x] をマーク
```

**完了基準**: 統合テストが成功

## Phase 5: 仕上げ

```
1. コードの重複を排除（DRY原則）
2. 命名を改善
3. ドキュメントコメントを追加
4. パフォーマンス最適化
5. tasks.md の [x] をマーク
```

**完了基準**: すべてのテストが成功し、コード品質が基準を満たす
