# フェーズ実行順序

```
Phase 1: 基盤構築（セットアップ）
   |
Phase 2: コア実装（TDDループ）
   |
Phase 3: 統合
   |
Phase 4: テスト
   |
Phase 5: 仕上げ（ポリッシュ）
```

## 開始フェーズの判定

```
- Phase 1 の全タスク完了 -> Phase 2 から開始
- Phase 2 の全タスク完了 -> Phase 3 から開始
- 以降同様...
```

**--phase オプション指定時**: 指定されたフェーズから強制開始

## フェーズごとの実行

````
各フェーズについて:

1. フェーズのタスク一覧を表示
2. 各タスクを順番に実行
   a. タスク詳細を確認
   b. 設計書・仕様書を参照して実装
   c. TDDアプローチに従う
   d. 完了後に tasks.md の [x] をマーク
3. フェーズ完了後にテストを実行
4. すべてのテストが成功したら次のフェーズへ
````
