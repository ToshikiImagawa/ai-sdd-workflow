# 継続検証

## 各タスク完了後の自動チェック

```
1. 関連テストを実行 -> 成功必須
   |
2. 仕様整合性チェック -> 一致必須
   |
3. tasks.md を更新 -> [x] をマーク
```

## 仕様整合性チェック（自動）

````markdown
### 仕様整合性チェック

| 検証項目                         | ステータス | 備考                        |
|:-------------------------------|:---------|:---------------------------|
| API署名が仕様と一致              | pass     | すべての公開APIが実装済み      |
| データモデルが仕様と一致          | pass     | 型が仕様と整合               |
| 動作がシーケンス図と一致          | pass     | フロー検証済み               |
| 非機能要件を満たしている          | warn     | レスポンスタイムの調整が必要   |
````
