# TDDサイクル（RED-GREEN-REFACTOR）

## 各タスクのTDDサイクル

```
1. RED: 失敗するテストを作成
   - 期待する動作を定義
   - テストが失敗すること（実装がまだないため）
   - コミット: "test: {feature} のテストを追加"

2. GREEN: テストを成功させる
   - テストを通す最小限のコードを作成
   - テストが成功することを確認
   - コミット: "feat: {feature} を実装"

3. REFACTOR: コードを改善
   - 動作を変えずに構造を改善
   - すべてのテストが引き続き成功することを確認
   - コミット: "refactor: {feature} をクリーンアップ"
```
